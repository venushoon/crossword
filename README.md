# crossword
# 🧩 십자말풀이 퍼즐 메이커 프로 (Crossword Maker Pro)

> **교사를 위한 정밀한 수동 에디터와 학생들을 위한 실시간 온라인 풀이 모드를 지원하는 단일 파일(Single File) 기반 십자말풀이 생성 웹앱입니다.**
> 웹 서버나 데이터베이스(DB) 연결 없이도 브라우저만 있으면 언제 어디서나 퍼즐을 설계하고, 학습지로 인쇄하거나 QR코드로 바로 배포할 수 있습니다.

---

## ✨ 핵심 특징 (Key Features)

* **서버 없는 무한 온라인 공유 (LZ-String 압축)**: 별도의 로그인이나 데이터베이스 없이, 선생님이 만든 퍼즐 데이터가 고도로 압축되어 URL 주소 자체에 내장됩니다. 
* **100% 인식률의 대형 QR 코드**: 데이터 압축 기술을 통해 QR 코드의 밀도를 대폭 낮췄습니다. 전자칠판에 띄워두면 멀리 앉은 학생들의 스마트폰이나 태블릿 카메라도 즉시 인식합니다.
* **전문가용 대시보드 레이아웃**: 와이드 모니터 및 전자칠판 환경에 최적화된 사이드바 메뉴 방식을 채택하여 넓은 작업 공간을 확보했습니다.
* **단일 HTML 구성**: 다른 패키지나 서버 설치 없이 `index.html` 파일 하나만 GitHub Pages에 업로드하면 즉시 나만의 도메인으로 배포 및 서비스가 가능합니다.

---

## 🚀 주요 기능 (Main Functions)

### 👨‍🏫 교사 에디터 모드 (Teacher Dashboard)
* **정밀한 수동 배치 기능**: 모눈종이 캔버스를 직접 클릭하여 단어의 시작 좌표, 방향(가로/세로)을 선택해 십자말을 조립할 수 있습니다. 이미 배치된 글자와 자연스럽게 겹치며 교차하는 크로스워드 로직이 완벽하게 작동합니다.
* **환경 설정 옵션 패널**: 
  * 퍼즐의 단원명이나 학습 목표를 지정하는 **주제(제목) 입력 기능**
  * 학급 규모나 단어 수에 맞춰 유연하게 조절하는 **그리드 칸 수 설정 (가로/세로)**
* **디자인 템플릿 전환**:
  * **전통 🟦**: 빈칸의 색상(배경색)을 자유롭게 커스텀하여 꽉 찬 퍼즐판 렌더링
  * **선 날리기 ✨**: 빈칸의 배경을 완전히 투명하게 날려 단어 상자들만 세련되게 띄우는 최신 트렌드 디자인
* **A4 인쇄 최적화**: 인쇄(출력) 버튼 클릭 시 에디터 UI는 깔끔하게 숨겨지며, 지정한 제목, 퍼즐판, 가로/세로 열쇠 목록이 A4 용지 규격에 딱 맞춰 정렬되어 학습지로 출력됩니다.
* **브라우저 임시 보관함**: `Local Storage` 기술을 활용하여 로그인 없이도 작업 중인 퍼즐을 무제한으로 저장하고 언제든 불러와 다시 수정하거나 재활용할 수 있습니다.

### 🧑‍🎓 학생 온라인 풀이 모드 (Student Mode)
* **글자 가운데 정렬 및 전용 UI**: 공유받은 QR/URL로 접속하면 입력창이 없는 깔끔한 학생용 화면으로 진입합니다. 모든 디바이스에서 글자가 정중앙에 예쁘게 정렬됩니다.
* **실시간 자동 채점**: 학생들이 칸을 채운 후 `정답 확인` 버튼을 누르면, 맞은 글자는 파란색 배경(`#EFF6FF`), 틀린 글자는 빨간색 배경 (`#FEF2F2`)으로 시각적 피드백을 즉시 제공합니다.
* **구조화된 힌트 목록**: 화면 하단에 가로 열쇠와 세로 열쇠가 번호 순으로 깔끔하게 정렬되어 있어 가독성이 뛰어납니다.

## 🛠 사용된 기술 스택 (Tech Stack)

* **Frontend**: React (v18, CDN 방식)
* **Styling**: Tailwind CSS (v3)
* **Compiler**: BabelStandalone (실시간 인브라우저 트랜스파일링)
* **Compression**: LZ-String (URL 인코딩 및 데이터 압축)
* **Icons**: Inline SVG Icons

## 📦 설치 및 실행 방법 (Deployment)

이 프로젝트는 빌드 과정이 필요 없는 독립형 단일 HTML 파일로 구성되어 있습니다.

1. 이 저장소의 `index.html` 코드를 복사합니다.
2. 본인의 GitHub에 새 저장소(Repository)를 개설합니다.
3. `index.html` 파일을 생성하고 코드를 붙여넣은 뒤 커밋(Commit)합니다.
4. 저장소의 **Settings > Pages** 메뉴에서 Source를 `Deploy from a branch`, Branch를 `main` (또는 `master`)으로 설정 후 저장합니다.
5. 제공되는 깃허브 페이지 링크를 통해 즉시 접속하여 사용할 수 있습니다.
